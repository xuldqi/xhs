# 🎉 专业文档格式功能 - 本地测试指南

## ✅ 开发服务器已启动

服务器地址：**http://localhost:5174/**

## 📝 测试步骤

### 1. 访问应用
打开浏览器，访问：http://localhost:5174/

### 2. 生成指南
1. 点击"开始分析"
2. 填写账号信息（可以使用测试数据）：
   - 账号名：测试账号
   - 粉丝数：10000
   - 笔记数：150
   - 内容类别：美妆
3. 点击"生成指南"
4. 等待指南生成完成

### 3. 测试格式切换
指南生成完成后，你应该能看到：

#### ✨ 新增的格式选择器
在指南顶部，操作按钮下方，会有一个格式选择器：
- 📱 **卡片格式** - 原有的彩色卡片布局
- 📄 **专业文档** - 新的专业文档格式

#### 🔄 测试切换功能
1. **点击"专业文档"按钮**
   - 应该看到页面切换到白色背景的专业文档格式
   - 包含封面页（标题、账号信息、日期）
   - 包含目录页（可点击跳转）
   - 内容以专业文档样式显示

2. **点击"卡片格式"按钮**
   - 应该切换回原来的彩色卡片格式
   - 折叠面板样式

3. **刷新页面**
   - 应该保持你最后选择的格式（localStorage 持久化）

### 4. 测试专业文档功能

#### 📄 封面页
- 显示"小红书账号增长策略报告"标题
- 显示账号名称
- 显示粉丝数、笔记数、类别
- 显示生成日期

#### 📑 目录页
- 列出所有章节
- 点击章节标题应该跳转到对应内容

#### 📝 内容章节
- 章节标题带自动编号（1. 2. 3.）
- 段落格式化
- 列表显示
- 表格样式（如果内容中有表格）

#### 🎨 样式检查
- A4 尺寸布局（白色背景）
- 专业的字体和间距
- 清晰的层级结构
- 页脚信息

### 5. 测试移动端
1. 打开浏览器开发者工具（F12）
2. 切换到移动设备模拟
3. 检查专业文档格式在移动端的显示
4. 应该自动调整为移动端布局

## 🐛 可能遇到的问题

### 问题 1：格式选择器不显示
**原因**：可能是组件导入问题
**解决**：检查浏览器控制台是否有错误

### 问题 2：切换格式没有反应
**原因**：可能是状态管理问题
**解决**：
1. 打开浏览器控制台
2. 检查是否有 JavaScript 错误
3. 查看 localStorage 中是否保存了 'preferredFormat'

### 问题 3：专业文档样式不正确
**原因**：CSS 文件可能没有正确加载
**解决**：
1. 检查浏览器控制台的 Network 标签
2. 确认 professional-document.css 已加载
3. 刷新页面

### 问题 4：账号数据不显示
**原因**：accountData 可能没有正确传递
**解决**：
1. 检查浏览器控制台
2. 查看 Vue DevTools 中的组件数据
